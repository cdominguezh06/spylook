### Uso de Optional y tipos no-nulos
En Kotlin, aunque se pueda usar la clase Optional igual que en Java, se opta por utilizar la verificaciÃ³n de valores nulos (null safety)

Todos los valores son por defecto no-nulos, es decir, se requiere asignar un valor al momento de la declaraciÃ³n y este nunca puede ser nulo
Mediante el uso de palabras clave como "lateinit" delante del nombre de la variable permiten separar declaraciÃ³n de inicializaciÃ³n solo en objetos (los tipos primitivos no son compatibles con esta palabra clave)

Esto introduce un nuevo requerimiento a la hora de usar la variable, asegurarse de que al momento de acceder a la variable no es nula
